a, button, input 요소

a
                                        경로가 지정된 버튼
                                        경로가 지정된 버튼
  • - 링크/버튼요소에는 반드시 텍스트를 사용해야 한다.
  • - href속성은 생략을 금지한다.
  • - 문서간 연결이 되어있는 버튼은 a태그를 사용한다
button
  • - 클릭함으로서 이벤트가 발생하는 것은 button태그를 사용한다.
  • - type을 명시하지 않으면 submit으로 실행하기 때문에 type="button"으로 지정해야 한다.
  • - button type="submit" 은 폼을 전송하는 기능을 담당한다. [default]
  • - button type="reset" 은 폼에 입력한 내용을 초기화 하는 기능을 담당한다.
  • - button type="button"은 js로 기능구현 인터페이스를 조작할 수 있다. 디자인적 관점에서도 input요소와 달리 button은 자유롭다. button태그 사이에 다른 태그들을 삽입 가능하다.
input
  • - 사용자의 입력을 서버로 전송하기 위한 버튼으로 전달하는 값을 받거나 전달할 때 사용한다.
  • - input태그는 가상선택자(::before, ::after)를 사용할 수 없다.

주석
  • - 시작과 끝의 표시는 //로 구분한다.
코드 들여쓰기
  • - 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.
빈줄
  • - 가독성을 고려하여 빈 줄을 추가한다. 단, 1줄을 초과하지 않는다.
마크업 유의사항
  • - p 태그 안에는 p, div, ul 등 다른 블럭요소가 들어갈 수 없다. ( display:inline 속성을 적용해도 소용없음)
  • - p 태그는 문단을 구분하는 용도로만 사용하고 영역을 구분하는 용도로 사용하지 말 것.
  • - a 태그에 삽입된 img 태그에는 이미지맵을 사용할 수 없다.
  • - a 태그 안에는 a태그가 들어갈 수 없다.
  • - form 태그 안에는 form태그가 들어갈 수 없다.
  • - label 태그 안에는 label태그가 들어갈 수 없다.
  • - pre 태그 안에는 img, object, big, small, sub, sup태그가 들어갈 수 없다.
  • - button 태그 안에는 input, select, textarea, label, button, form, fieldset, iframe, isindex태그가 들어갈 수 없다.
  • - button 태그의 type 속성을 지정하지 않았을 때 IE6, IE7 의 경우 submit 기능으로 동작하므로 폼 값을 전송하는 용도로 사용하지 않을 경우 type="button"을 지정
  • - 태그의 중첩에 유의
    <em><strong>내용</em></strong> (X)
    <em><strong>내용</strong></em> (O)
  • - 하나의 ID는 페이지에서 한번만 사용할 것 (스타일은 적용되나 책갈피 기능이나 스크립트를 정상적으로 사용할 수 없음)
